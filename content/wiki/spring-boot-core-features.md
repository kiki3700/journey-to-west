## 1. Opinionated (의견이 반영된 설정)

> 기존의 스프링 프레임워크는 라이브러리에 대한 높은 자유도를 제공했지만, 그만큼 설정의 복잡성과 호환성 문제가 발생하여 초기 개발 난이도가 높았습니다. 이에 반해 스프링 부트는 "좋은 기본값"을 제공하여 개발자가 비즈니스 로직에 집중할 수 있도록 돕습니다.

- **자기 주장이 강한 프레임워크 철학**을 기반으로 설계됨
- 스프링 부트가 **의존성 구성**, **설정**, **기본 동작**을 버전에 맞춰 자동으로 처리
- `spring-boot-starter-*` 의존성만 추가해도 관련 라이브러리와 설정이 자동으로 구성됨
- 기본값은 제공하지만, 필요에 따라 **유연하게 커스터마이징 가능**

---

## 2. Embedded Server (내장 서버)

> `main()` 메서드 실행만으로 웹 애플리케이션을 구동할 수 있습니다. 이는 Python의 Flask, Node.js의 Express 등 다른 인기 프레임워크들과 동일한 접근 방식으로, 단일 실행 파일만으로 서버를 띄울 수 있어 배포 환경 설정이 단순하고 개발 및 테스트가 쉬워집니다. 스프링 부트도 이러한 방식을 채택하여, 별도의 웹 서버 설치나 복잡한 설정 없이 애플리케이션을 바로 실행할 수 있도록 지원합니다.

- 전통적인 스프링 애플리케이션은 `.war` 파일을 만들어 외부 Tomcat 서버에 배포해야 했음
- 스프링 부트는 **내장 Tomcat, Jetty, Undertow** 등을 포함하여 `.jar` 파일만으로 실행 가능
- `java -jar myapp.jar` 한 줄로 실행 가능 → 운영 및 배포 절차가 간소화됨

---

## 3. Auto Configuration (자동 설정)

> 클래스패스에 존재하는 라이브러리를 기준으로, 관련 설정을 자동으로 적용합니다.

- `@EnableAutoConfiguration` 애노테이션(→ `@SpringBootApplication`에 포함됨)을 통해 작동
- 예를 들어 `spring-boot-starter-web`을 추가하면:
  - DispatcherServlet 자동 등록
  - 내장 Tomcat 자동 설정
  - Jackson을 활용한 JSON 변환 설정 등 자동 적용
- 설정 파일을 수동으로 작성하지 않아도 빠르게 애플리케이션을 실행 가능

---

## 📝 총평

> 스프링 부트는 **opinionated한 자동 설정**과 **내장 서버 기반 실행**을 통해,
> 개발자가 빠르게 애플리케이션을 만들고 배포할 수 있는 **단순하고 효율적인 개발 경험**을 제공합니다.
